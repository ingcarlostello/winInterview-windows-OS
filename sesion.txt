Key Decisions
- Hosted checkout over embedded Paddle.js: more reliable with Tauri CSP/webview; checkout opens in system browser via open_url Rust command
- Lite = paid tier: New users default to free (180s transcription, was 0 captures/analyses → pending change to 1/1); updateUserPlan converted to internalMutation (was public security vulnerability)
- Webhook on Convex HTTP actions: Consistent with existing Clerk webhook pattern; no backend Python changes needed
- custom_data.plan_id on price as the mapping mechanism: Paddle propagates to subscription events; fallback to SANDBOX_PRICE_IDS map in paddle.ts
- custom_data.clerk_id on transaction: Links Paddle customer to Clerk user; passed through createCheckout action from authenticated identity
- No feature flags needed for single capture/analysis: canCaptureScreen() allows 1 capture without simultaneous_captures; backend only gates SIMULTANEOUS_ANALYSIS when len(images) > 1
