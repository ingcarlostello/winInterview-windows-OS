Garbage Collector en JavaScript:

El GC usa el algoritmo mark-and-sweep. Los problemas ocurren con:

  • Closures que capturan variables innecesariamente
  • Event listeners no removidos
  • Referencias circulares no resueltas

class RateLimiter { constructor(redis, limit) { this.redis = redis; this.limit = limit; } async check(userId) { const key = `rate:${userId}`; const count = await this.redis.incr(key); if (count === 1) { await this.redis.expire(key, 60); } return count <= this.limit; } }
